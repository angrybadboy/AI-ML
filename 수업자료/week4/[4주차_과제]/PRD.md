## 4. 기능 요구사항

### 4.1 Tab 1 — 1D 함수 근사 (Universal Approximation 증명)

| ID | 기능 | 설명 |
|----|------|------|
| F1-01 | 함수 선택 | sin(x), cos(x)+0.5sin(2x), x·sin(x), 복합함수 |
| F1-02 | 네트워크 크기 선택 | Small/Medium/Large/Very Large |
| F1-03 | 활성화 함수 선택 | tanh / relu |
| F1-04 | Epochs 설정 | 200 ~ 5000 |
| F1-05 | 결과 시각화 | 함수 근사 / Loss 곡선 / 절대 오차 분포 |
| F1-06 | 수치 결과 | MSE, MAE, 최대 오차 표시 |

**증명 포인트:** 네트워크가 클수록 MSE가 감소하며 Universal Approximation Theorem이 시각적으로 증명됨

---

### 4.2 Tab 2 — 포물선 운동 회귀 (인터랙티브)

| ID | 기능 | 설명 |
|----|------|------|
| F2-01 | 모델 학습 | 2000개 물리 데이터로 NN 학습 (100 epochs) |
| F2-02 | v₀ 슬라이더 | 초기 속도 10~50 m/s 실시간 조정 |
| F2-03 | θ 슬라이더 | 발사 각도 20~70° 실시간 조정 |
| F2-04 | 실시간 예측 | 슬라이더 변경 즉시 궤적 그래프 업데이트 |
| F2-05 | 물리 결과 | 최대 높이, 최대 거리, MSE 표시 |
| F2-06 | 3종 그래프 | 2D 궤적 / x(t) / y(t) 동시 표시 |

**증명 포인트:** NN이 `x(t)=v₀cosθ·t`, `y(t)=v₀sinθ·t-½gt²` 물리 공식을 데이터에서 학습함

---

### 4.3 Tab 3 — 과적합 vs 과소적합 (3모델 동시 비교)

| ID | 기능 | 설명 |
|----|------|------|
| F3-01 | 노이즈 수준 조정 | 0.05 ~ 1.0 |
| F3-02 | 학습 데이터 수 조정 | 30 ~ 300개 |
| F3-03 | Epochs 조정 | 50 ~ 500 |
| F3-04 | 3모델 동시 학습 | Underfit / Good Fit / Overfit |
| F3-05 | 예측 비교 | 상단 행: 3모델 예측 vs 실제 함수 |
| F3-06 | Loss 곡선 비교 | 하단 행: Train/Val Loss 곡선 (log 스케일) |

**증명 포인트:**
- Underfit: Train/Val 모두 높음 → 모델이 너무 단순
- Good Fit: Train/Val 비슷하게 낮음 → 이상적
- Overfit: Train 낮음, Val 높음 → 노이즈까지 학습

---

### 4.4 Tab 4 — 진자 주기 예측 + RK4 시뮬레이션 (인터랙티브)

| ID | 기능 | 설명 |
|----|------|------|
| F4-01 | 모델 학습 | 2000개 진자 데이터로 NN 학습 (200 epochs) |
| F4-02 | L 슬라이더 | 진자 길이 0.5~3.0 m 실시간 조정 |
| F4-03 | θ₀ 슬라이더 | 초기 각도 5~80° 실시간 조정 |
| F4-04 | 주기 vs 각도 | NN 예측 vs 물리 공식 비교 그래프 |
| F4-05 | 주기 vs 길이 | √L 비선형 관계 시각화 |
| F4-06 | RK4 시뮬레이션 | d²θ/dt²=-(g/L)sinθ 수치 적분 |
| F4-07 | 위상 공간 | 각도 vs 각속도 위상 다이어그램 |
| F4-08 | 수치 결과 | NN 예측 주기 / 실제 주기 / 오차(%) |

**증명 포인트:**
- 작은 각도: 등시성 (T ≈ T₀ = 2π√(L/g)) 확인
- 큰 각도: 주기 증가 현상 확인
- L ↑ 4배 → T ↑ 2배 (√L 관계) 시각화
