## 1. 제품 개요

### 1.1 Problem

1. **소비는 많지만 "간직"되지 않는다.** SNS에서 마음에 드는 글귀를 만나도 스크린샷 폴더에 방치되거나, 피드 흐름 속에 휘발된다.
2. **몰입 독서의 장벽이 높다.** 긴 아티클은 부담스럽고, 짧은 글은 광고·알고리즘에 의해 집중이 깨진다.
3. **쓰고 싶은 사람에게 가벼운 공간이 없다.** 블로그/인스타는 노출 부담이 크고, 글귀 단위로 쓰기엔 과한 도구다.

### 1.2 Vision

> "하루 3~5분, 한 편의 글을 나만의 속도로 읽고 간직하는 공간"

소비가 아니라 **리추얼**이 되는 서비스. 방문 → 읽기 → 저장/감상 → 떠남이라는 사이클이 짧고 명확하다.

### 1.3 Goals

| 유형 | 목표 |
|---|---|
| **Primary (User Goal)** | 사용자가 매일 1회 앱에 방문해, 오늘의 글 한 편을 온전히 읽고 떠나게 한다. |
| **Secondary (Engagement)** | 저장·감상·작성·공유 중 1개 이상의 행동이 세션당 발생하도록 한다. |
| **Academic (학습 목표)** | 인증, 결제(샌드박스), AI 연동, 커뮤니티 4가지 축을 풀스택으로 구현·증빙한다. |

### 1.4 Non-goals

- ❌ 실결제 수익화 (샌드박스 한정)
- ❌ 긴 아티클 / 에세이 / 뉴스레터 플랫폼
- ❌ 영상·오디오 콘텐츠
- ❌ 다국어 지원 (한국어 only)
- ❌ 실시간 채팅·DM 기능
- ❌ 팔로우 기반 소셜 그래프 (과제 스코프 외)

## 2. 타겟 사용자

### 2.1 Primary Persona — 감정 정돈러, 지연(28세, 마케터)

- 출퇴근길·자기 전 짧은 시간에 "마음 환기용" 콘텐츠를 찾는다
- Instagram에서 글귀 계정을 팔로우하며, 스크린샷 앨범이 1,000장+
- 시각적으로 예쁜 것에 반응하고, 거친 UI엔 금방 흥미를 잃는다
- 결제 경험: Melon, Netflix 등 월 구독에 익숙

**핵심 니즈**: "마음에 든 글을 다시 찾을 수 있는 곳이면 좋겠다."

### 2.2 Secondary Persona — 조용한 창작자, 민호(24세, 대학생)

- 블로그·SNS는 노출 부담 때문에 비활성
- 메모앱에 혼자 쓰는 문장이 쌓여 있음
- 익명 혹은 가벼운 정체성으로 자기 표현을 하고 싶음

**핵심 니즈**: "부담 없이 짧게 쓰고, 반응이 있으면 더 좋겠다."

## 3. 사용자 여정 (User Journey)

```
랜딩 → 오늘의 글귀 미리보기 → 회원가입 → 오늘의 글(읽기) → 저장 → 마이페이지
                                                       ↘ 감상 작성
                                                       ↘ 다른 글 더 보기 (무료 한도 초과 시 결제 유도)
                                                       ↘ 글귀 작성 → 공유
```

### 3.1 핵심 시나리오 — "하루의 글 읽기"

1. 앱 진입 시 **오늘의 글귀**가 즉시 노출 (로딩 0.5초 이내 첫 화면)
2. 문장이 페이드 인으로 서서히 드러남 — 광고·알림 없음
3. 스크롤 또는 탭으로 다음 단락 전개 (카드형 또는 스토리형)
4. 글 끝에서 세 가지 옵션: **저장 / 감상 남기기 / 다른 글 보기**
5. 하루 1편 무료, 추가 탐색은 프리미엄 유도

### 3.2 부가 시나리오 — "내가 쓴 글 공유"

1. 마이페이지 → `글귀 쓰기`
2. 미니멀 에디터 (제목 / 본문 / 태그)
3. 공개 / 비공개 설정
4. 공개 글은 `발견` 피드에 노출 (다른 사용자가 볼 수 있음)

## 4. 기능 요구사항

### 🔴 P0 — MVP 필수

| # | 기능 | 상세 |
|---|---|---|
| F-01 | **이메일 회원가입/로그인** | Supabase Auth 기반, 이메일 인증 |
| F-02 | **소셜 로그인** | Google OAuth 1종 (솔로 개발 스코프 고려) |
| F-03 | **오늘의 글귀 제공** | AI 생성(Claude API) + 큐레이션 혼합. 하루 1편 고정 |
| F-04 | **글 저장(북마크)** | 사용자별 저장 목록 |
| F-05 | **마이페이지 — 저장한 글귀** | 저장 리스트, 썸네일/제목, 삭제 |
| F-06 | **마이페이지 — 내가 쓴 글귀** | 작성·수정·삭제 |
| F-07 | **글귀 작성 에디터** | 제목 ≤ 40자, 본문 ≤ 600자, 공개/비공개 |
| F-08 | **발견 피드 (공유된 글)** | 최신순, 좋아요 수 기준 간단 정렬 |
| F-09 | **결제 플로우 (샌드박스)** | Toss Payments 테스트 키 기반 정기결제 시뮬레이션 |
| F-10 | **프리미엄 진입 유도** | 무료 하루 1편 소진 시 결제 유도 모달 |

| # | 기능 | 상세 |
|---|---|---|

| F-12 | **좋아요 / 공감 리액션** | 발견 피드용 |

## 5. UX/UI 원칙 — 감성의 구체화

감성적 UI는 추상어다. 실제로 구현 가능한 원칙으로 분해한다.

### 5.1 타이포그래피 퍼스트

- 글꼴: 본문 세리프 (예: `Noto Serif KR`, `IBM Plex Serif`), 포인트는 산세리프
- 본문 크기: 모바일 18~20px, 줄간격 1.7
- 글자 하나하나가 주인공이 되도록, 여백이 압도적으로 넓어야 함

### 5.2 모션 — "드러남"의 연출

- 글귀 페이드 인: 0.8초 ease-out
- 스크롤 시 문단 단위 등장 (Framer Motion `whileInView`)
- 저장 인터랙션: 하트가 잉크처럼 번지는 애니메이션

### 5.3 색감 — 무채색 기반 + 감정 포인트

- 베이스: 오프화이트 / 크림 / 차콜
- 포인트: 글귀 카테고리별 1개 컬러 (최대 5색 팔레트)
- 광택 없는 매트 질감, 그림자는 부드러운 글로우 계열

### 5.4 사운드·알림 — 의도적 부재

- 소리 없음, 팝업 알림 없음, 토스트도 최소화
- "조용한 방"의 감각을 해치지 않는 것이 핵심

### 5.5 레이아웃 — 한 화면 한 메시지

- 한 스크린에 하나의 행동만 제시
- 메뉴는 최소화 (홈 / 발견 / 쓰기 / 마이)

## 6. 성공 지표

### 6.1 North Star Metric

**DAU 중 "오늘의 글을 끝까지 읽은 사용자 비율"** (Read-through Rate)

### 6.2 Funnel KPI

| 단계 | 지표 | 목표(과제 시연 기준) |
|---|---|---|
| Acquisition | 가입 완료율 | 랜딩 방문자 중 30% |
| Activation | 첫 세션에서 글 1편 완독 | 가입자의 70% |
| Retention | D+1 재방문 | 40% |
| Action | 저장/작성/공유 1회 이상 | 세션당 50% |
| Revenue(Sandbox) | 결제 플로우 도달 → 샌드박스 결제 완료 | 도달자의 60% |

> 과제 시연에선 실수치 시뮬레이션(mock event)으로 funnel을 시각화할 수 있다.

### 6.3 과제 평가 관점의 지표

- 기능 완성도: P0 10개 중 몇 개 작동하는가
- 기술 스택 활용도: 인증/결제/AI/DB 각 축 증빙
- UI 완성도: 감성 UI 원칙 5개 중 구현된 것
- 문서 품질: PRD / TRD / README

## 7. 릴리즈 계획

### Phase 1 — Foundation (1주차)
- 프로젝트 셋업, 디자인 토큰 확정, Supabase 연동
- 로그인/회원가입, 기본 레이아웃
- **Exit Criteria**: 로그인한 사용자가 빈 홈 화면을 볼 수 있다

### Phase 2 — Core Reading (2주차)
- Claude API 연동, 오늘의 글귀 생성 파이프라인
- 읽기 화면 모션·타이포 완성
- 저장 기능
- **Exit Criteria**: 매일 새로운 글이 노출되고 저장·재조회가 된다

### Phase 3 — Write & Share (3주차)
- 글귀 작성 에디터
- 발견 피드
- 마이페이지
- **Exit Criteria**: 사용자가 글을 쓰고 공유하고 다른 글을 볼 수 있다

### Phase 4 — Payment Sandbox (4주차)
- Toss Payments 샌드박스 연동
- 프리미엄 유도 모달
- 구독 상태 관리
- **Exit Criteria**: 테스트 카드로 정기결제 시뮬레이션이 동작한다

### Phase 5 — Polish & 문서화 (5~6주차)
- 감성 UI 마감 (모션·폰트·여백)
- 버그 수정
- README, 발표자료, 시연 영상
- **Exit Criteria**: 제출 가능한 상태

## 8. 제약사항 & 가정

### 8.1 제약 (Constraints)
- 솔로 개발 / Claude Code 단독 구현
- 실결제 불가 → 샌드박스 전용
- 모바일 퍼스트 반응형 (별도 네이티브 앱 없음)
- 콘텐츠 저작권 — AI 생성 또는 사용자 작성만 사용 (외부 인용 금지)

### 8.2 가정 (Assumptions)
- Claude API 비용은 학생 크레딧 범위 내에서 충당 가능
- Supabase 무료 티어가 과제 기간 트래픽을 감당함
- 시연 기간 동시접속은 20명 미만
